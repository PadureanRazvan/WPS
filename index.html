Roadmap

Bugs Cunoscute

  • Gemini API returnează 404 pe unele browsere din cauza cache-ului — necesită Ctrl+Shift+R ✅ Rezolvat
  • Rata de cereri API limitată fără profil de facturare Google configurat
  • Cross-Origin-Opener-Policy blochează popup-ul de login pe unele browsere
  • Gemini 3.x „thinking mode" activ implicit provoacă eroare 400 cu function calling ✅ Rezolvat
  • Function response trimis ca array în loc de obiect — eroare proto serialization ✅ Rezolvat
  • Acțiunile de editare din planner nu sunt înregistrate în jurnal

Constatări & Îmbunătățiri Recomandate

  • Accesibilitate — lipsesc atribute ARIA, navigare cu tastatura, contrast insuficient pe text secundar
  • Performanță — tabelul planner-ului generează 2000+ noduri DOM; ar beneficia de virtualizare
  • Securitate — cheia Gemini API este expusă client-side; ar trebui proxy prin Cloud Functions
  • Stări de încărcare — lipsesc spinners/skeletons la încărcarea datelor din Firestore
  • Validare date — lipsește validare de schemă la scrierea în Firestore
  • Istoric chat — se pierde la reîncărcarea paginii (doar în memorie)
  • AI primea toate datele în prompt (10KB+) — migrat la Gemini function calling ✅ Implementat
  • Vulnerabilitate XSS în renderizarea mesajelor chat ✅ Rezolvat
  • AI putea șterge agenți fără confirmare utilizator ✅ Adăugat gate de confirmare

Dezvoltări Viitoare

  • Persistența istoricului chat în localStorage
  • Export rapoarte în PDF / Excel
  • Notificări push pentru schimbări de program
  • Roluri utilizator (admin, manager, vizualizare)
  • Dashboard cu grafice interactive (drag-and-drop pe planner)
  • Integrare calendar (Google Calendar sync)
  • Mod offline cu sincronizare la reconectare
  • Audit trail complet cu revert la versiuni anterioare ✅ Jurnal activitate implementat
  • Logare acțiuni planner (editare celule, creare/ștergere agenți) în jurnal
  • Proxy API key prin Cloud Functions pentru securitate
  • Point-in-Time Restore — salvare automată a snapshot-urilor plannerului în Firestore, cu posibilitatea de a restaura starea echipei la orice moment din trecut

Informații

1. Dashboard

Pagina principală oferă o vedere de ansamblu a echipei:

  • Total Agenți Activi — numărul de agenți înregistrați și activi în sistem
  • Ore Planificate — totalul orelor planificate pentru ziua curentă (Azi) sau orice dată selectată din calendar
  • Ore Alocate pe Echipe — tabelul arată distribuția orelor pe echipe pentru data selectată
  • Productivitate Medie — media productivității (tickete+apeluri / oră) pe ultimele zile cu date încărcate
  • Trend Productivitate — graficul arată productivitatea per echipă (tickete+apeluri / oră) pe toată luna curentă

2. Users (Utilizatori)

Gestionarea agenților din echipă:

  • Adaugă agent nou — completează numele, username-ul, orele de contract, tipul contractului, echipa primară și data angajării
  • Editare inline — click direct pe orice câmp din tabel pentru a-l modifica (numele, orele, echipa etc.)
  • Echipa primară — echipa principală a agentului (ex: RO zooplus). Agentul poate lucra și pentru alte echipe conform plannerului
  • Tip Contract — Full-time (8h automat) sau Part-time (ore configurabile). Schimbarea tipului de contract deschide un modal pentru a selecta data de start și noile ore
  • Activ/Inactiv — buton care deschide un modal cu opțiuni: Indefinit (dezactivare permanentă) sau Perioadă (interval de date). Plannerul se actualizează automat

3. Planner

Planificarea lunară a orelor de lucru per agent:

  • Format ore — introdu orele cu codul echipei: 8RO = 8 ore pe RO, 4RO+4IT = 4 ore RO + 4 ore IT
  • Coduri speciale — Co = Concediu, CM = Concediu Medical, SL = Sărbătoare Legală, LB = Liber
  • Selecție multiplă — selectează mai multe celule și editează-le simultan cu modalul de editare
  • Salvare automată — toate modificările se salvează automat în baza de date

4. Productivitate

Analiza productivității bazată pe datele încărcate (tickete + apeluri) și orele din planner:

  • Selectare perioadă — alege un interval de date cu calendarul din colțul dreapta-sus
  • Filtru echipă — selectează o echipă specifică (ex: RO, IT) pentru a vedea doar ticketele și apelurile rezolvate pentru acea echipă
  • Sumar — vedere agregată per agent pe întreaga perioadă selectată
  • Per Agent — vedere detaliată zi cu zi, per agent. Selectează agenții folosind chipurile sau căutarea
  • Echipe (coloana) — arată echipele pentru care agentul a lucrat efectiv, bazat pe fișierele încărcate
  • Formula — Productivitate = (Tickete + Apeluri) / Ore Lucrate

5. Upload File

Încărcarea fișierelor de productivitate zilnică:

  • Pasul 1: Selectează data — alege data din calendar pentru care încarci datele. Sistemul avertizează dacă data are deja date încărcate
  • Pasul 2: Fișier XLSX (Tickete) — încarcă fișierul Excel cu ticketele rezolvate. Sistemul extrage automat agentul, numărul de tickete și echipa
  • Pasul 3: Fișier CSV (Apeluri) — încarcă fișierul CSV cu apelurile. Sistemul extrage agentul, numărul de apeluri și echipa
  • Potrivire automată — numele din fișiere sunt normalizate și potrivite automat cu agenții din sistem
  • Persistență — datele sunt salvate în Firebase și se păstrează între sesiuni

6. Rapoarte

Rapoarte agregate bazate pe datele din planner:

  • Selectare interval — alege perioada dorită cu calendarul (default: luna curentă)
  • Ore Planificate per Shop — tabel cu totalul orelor planificate pe fiecare echipă și numărul de agenți
  • Distribuție Agenți per Shop — carduri per echipă cu fiecare agent și orele lui, ordonate descrescător

7. Sherpa AI

Asistent AI integrat care poate răspunde la întrebări și efectua acțiuni:

  • Întrebări — întreabă orice despre planificarea echipei, orele lucrate, productivitatea, sau distribuția pe echipe
  • Acțiuni — Sherpa AI poate modifica planificarea, adăuga sau șterge agenți, și naviga între pagini
  • Siguranță — acțiunile destructive (ștergere) necesită confirmare. Modificările sunt limitate și validate automat
  • Multilingv — răspunde în limba selectată (Română, Engleză, Italiană)
